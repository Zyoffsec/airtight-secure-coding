# Airtight — Gate Checklist

> One line per gate: the rule and its fix. Run this list before emitting code.

- Gate 1 — Auth response uniformity. Fix: Hash unconditionally. Compute one throwaway hash at startup with the same parameters as real ones, verify against it when the lookup misses, and collapse both outcomes into one 401 with one body. (OWASP A07, CWE-287)
- Gate 2 — Constant-time secret comparison. Fix: Compare with `crypto.timingSafeEqual` on equal-length buffers (Python: `hmac.compare_digest`). (OWASP A07, CWE-287)
- Gate 3 — Password storage. Fix: Hash with argon2id (`argon2-cffi`), or bcrypt where argon2 is unavailable. (OWASP A07, CWE-287)
- Gate 5 — Session lifecycle. Fix: Call the session store's regenerate on successful login and its destroy on logout, instead of assigning and clearing a `userId` field. (OWASP A07, CWE-287)
- Gate 6 — Identity token entropy. Fix: Use `crypto.randomBytes(32).toString("base64url")` (Python: `secrets.token_urlsafe(32)`). (OWASP A07, CWE-287)
- Gate 7 — Session cookie flags. Fix: Set all three: `httpOnly: true`, `secure: true`, `sameSite: "lax"`. (OWASP A07, CWE-287)
- Gate 8 — Reset token expiry and single use. Fix: Give the token row an `expiresAt` (an hour is generous) and a `usedAt`, reject when either check fails, and write `usedAt` in the same transaction as the password update so two concurrent redemptions cannot both land. (OWASP A07, CWE-287)
- Gate 10 — Server-side identity. Fix: Take identity only from `req.session` (or the server's verified token claims) and never read a user id from the request. (OWASP A01, CWE-284)
- Gate 11 — Privileged route authorization. Fix: Add a role check to the handler chain that reads the role from the session or from a server-side lookup by session user id, and reject with 403 before the handler body runs. (OWASP A01, CWE-284)
- Gate 12 — Object ownership check. Fix: Put the ownership column in the `WHERE` clause of the same query that fetches the record, so a non-owner gets zero rows rather than a record plus a check that a later refactor can drop. (OWASP A01, CWE-284)
- Gate 13 — Mass assignment allowlist. Fix: Declare a schema listing only the editable fields (a Pydantic model, a zod schema, a serializer) and write the validated object, never the raw body. (OWASP A01, CWE-284)
- Gate 15 — Tenant scoping. Fix: Resolve the tenant to the session user's membership before querying, and pass the resolved tenant id to the query rather than the path value. (OWASP A01, CWE-284)
- Gate 21 — Dynamic SQL identifiers. Fix: Look the value up in an object of literal strings written in the source, falling back to a default on a miss, and interpolate the looked-up literal. (OWASP A03, CWE-74)
- Gate 22 — SQL query construction. Fix: Pass values as bound parameters using the driver's placeholder syntax and let the driver do the quoting. (OWASP A03, CWE-74)
- Gate 24 — Query operator injection. Fix: Coerce at the boundary: `String(...)` for strings, `Number(...)` for numbers, and reject if the result is not what you expected. (OWASP A03, CWE-74)
- Gate 26 — OS command construction. Fix: Pass the command as an argument list with no shell (`shell=False` is the default; in Node, `execFile` or `spawn` without `shell: true`). (OWASP A03, CWE-74)
- Gate 28 — Runtime evaluation of user input. Fix: Keep the template source literal and pass the value as a named argument, so the engine parses markup you wrote and substitutes data it escapes. (OWASP A03, CWE-74)
- Gate 30 — Hardcoded credential literal. Fix: Read the value from the environment at the point of use and let the process fail if it is absent (Gate 31). (OWASP A05, CWE-798)
- Gate 31 — Secret fallback default. Fix: Index the environment directly (`os.environ["SECRET_KEY"]`, or a config schema that marks the field required) so a missing value stops the process at startup instead of substituting a known one. (OWASP A05, CWE-798)
- Gate 32 — Ignore rule before the file. Fix: Write the `.gitignore` rule in the same change that creates the file, before the file, with `!.env.example` so the placeholder template stays tracked. (OWASP A05, CWE-798)
- Gate 33 — Removing a committed secret. Fix: State that the key is compromised and name the revocation step at the issuer. (OWASP A05, CWE-798)
- Gate 35 — Secrets in the client bundle. Fix: Put the call behind a server route that holds the key server-side and forward the result. (OWASP A05, CWE-798)
- Gate 37 — Secrets in error responses. Fix: Return a literal message and a status code. Send the exception to the log instead — the log is an internal sink, the response body is not. (OWASP A05, CWE-798)
- Gate 40 — Server-side request validation. Fix: Parse the request at the top of the handler with a runtime schema (zod, Pydantic) and use its output. (OWASP A03, CWE-20)
- Gate 41 — Server-computed amounts. Fix: Accept only identifiers and quantities from the client, look up every price server-side by id, and compute the total in the handler. (OWASP A03, CWE-20)
- Gate 43 — Validated value is the one used. Fix: Use only the validator's output for the rest of the handler — `serializer.validated_data`, `parsed.data` — and never reference the raw request again after the parse. (OWASP A03, CWE-20)
- Gate 44 — Upload type from content. Fix: Sniff the bytes server-side (`file-type` in Node, `python-magic` or a Pillow `Image.open(...).verify()` in Python) and take both the accept decision and the stored extension from the sniff result, never from a client-supplied string. (OWASP A03, CWE-20)
- Gate 45 — Upload size cap. Fix: Set `limits.fileSize` and `limits.files` on the upload instance to the largest file the feature actually needs, so the stream aborts at the cap rather than after it. (OWASP A03, CWE-20)
- Gate 46 — Upload destination path. Fix: Generate the filename server-side — a UUID plus the extension from the Gate 44 sniff — write it to a directory outside every static root, and serve files through a handler that looks up the stored key. (OWASP A03, CWE-20)
- Gate 50 — Supplied HTML at a raw sink. Fix: Sanitize with DOMPurify at the sink — the last expression before the value enters the sink, in the same function. (OWASP A03, CWE-79)
- Gate 51 — Markup assembled by interpolation. Fix: Do not build the HTML. Set `textContent` on an element you created, or return a JSX child (`<li>{item.title}</li>`), or pass the value to a template placeholder — the escaping the framework already does is the thing being bypassed here. (OWASP A03, CWE-79)
- Gate 53 — URL scheme allowlist. Fix: Parse with `new URL(raw)` and test `url.protocol` against a `Set` of literal schemes, using the parsed `url.href` rather than the raw string. (OWASP A03, CWE-79)
- Gate 55 — Data in a script context. Fix: Serialize with a helper that escapes `<` — `serialize-javascript` in Node, `{{ value|json_script:"user-data" }}` in Django, `htmlEscapeJsonString` where a framework already ships one — or `JSON.stringify(v).replace(/</g, "\\u003c")` where none exists. (OWASP A03, CWE-79)
- Gate 60 — TLS verification. Fix: Supply the peer's CA as an explicit trust anchor — `new https.Agent({ ca })`, Python `requests.get(url, verify="/path/ca.pem")` or `httpx.Client(verify=ctx)`, Go `RootCAs` — so the one certificate you meant to accept is the only one added. (OWASP A02, CWE-327)
- Gate 61 — Unpredictable values from a CSPRNG. Fix: Draw from the CSPRNG: `crypto.randomInt(n)` per character where the value has a format to satisfy, or `crypto.randomBytes(32).toString("base64url")` where it does not (Python: `secrets.choice(alphabet)` / `secrets.token_urlsafe(32)`). (OWASP A02, CWE-327)
- Gate 63 — Cipher mode and nonce. Fix: Use `aes-256-gcm` with a 12-byte nonce from `crypto.randomBytes` generated *inside* the encrypt function on each call, and store `nonce || tag || ciphertext` as one blob — the nonce is not secret and needs no second column. (OWASP A02, CWE-327)
- Gate 65 — Encoding is not protection. Fix: Attach an HMAC-SHA256 of the value computed with a server-side key and verify it with `timingSafeEqual` (Gate 2) before acting, so the handler accepts only links the server issued. (OWASP A02, CWE-327)
- Gate 70 — Outbound destination allowlist. Fix: Parse the URL, check `protocol` and `hostname` against a `Set` of literal values written in the source, and reject otherwise — the allowlist is also what makes `file:///etc/passwd` and `gopher://` fail closed, since neither carries a host that can be on it. (OWASP A10, CWE-918)
- Gate 71 — Redirects on a fetched URL. Fix: Set `redirect: "manual"` (undici `maxRedirections: 0`, Python `allow_redirects=False`) and follow hops yourself, running the same scheme-and-host check on each `Location` before the next request, with a hop cap. (OWASP A10, CWE-918)
- Gate 74 — Stored callback destination. Fix: Send webhook traffic through a dedicated egress proxy that allows only public destinations, and give the delivery client that proxy as its dispatcher with redirects off. (OWASP A10, CWE-918)
- Gate 76 — Remote resources in rendered documents. Fix: Turn on request interception and abort by default, continuing only for a scheme and host on a literal allowlist — inline the document's own assets as data URIs so the render needs no network at all. (OWASP A10, CWE-918)
- Gate 80 — CORS origin allowlist. Fix: Enumerate the origins as literal strings in the source and let the middleware match the request against that list. (OWASP A05, CWE-16)
- Gate 81 — Debug mode in a deployed configuration. Fix: Start the app from a production server in the deployed configuration (`gunicorn`, `uvicorn` without `--reload`, `node` with `NODE_ENV=production` set in the image) and leave `app.run()` for the local dev command only. (OWASP A05, CWE-16)
- Gate 83 — Default account credential. Fix: Read the value from the environment with no fallback and let the seed refuse to run without it (Gate 31). (OWASP A05, CWE-16)
- Gate 85 — Datastore and admin surface exposure. Fix: Drop the `ports` entry for anything only other services talk to, and prefix the rest with `127.0.0.1:` so the binding is loopback-only. (OWASP A05, CWE-16)
- Gate 87 — Permissions on generated secret files. Fix: Pass the mode at creation — `{ mode: 0o600 }` in Node, `os.open(path, os.O_WRONLY | os.O_CREAT | os.O_TRUNC, 0o600)` wrapped in `os.fdopen` in Python. (OWASP A05, CWE-16)
- Gate 90 — Untrusted deserialization. Fix: Parse into a declared schema with a data-only format — `Workspace.model_validate_json` (Pydantic), `yaml.safe_load` where the format must be YAML — and let the parse fail closed on anything the schema does not name. (OWASP A08, CWE-345)
- Gate 91 — Third-party script integrity. Fix: Pin an exact version in the URL, add `integrity` with the SHA-384 digest of that exact file, and add `crossorigin="anonymous"` — without the attribute the browser makes an opaque no-CORS request it cannot hash, so the check is skipped while the tag still looks protected. (OWASP A08, CWE-345)
- Gate 93 — Code fetched then executed. Fix: Download to a file, verify with `sha256sum -c` against a digest written in the source, then execute. (OWASP A08, CWE-345)
- Gate 95 — Update and plugin signature. Fix: Sign the release artifact in the release process with an ed25519 key, ship the public half in the bundle, and verify the downloaded bytes against it before they are written — an artifact written first is already sitting in the path the loader reads. (OWASP A08, CWE-345)
- Gate 96 — Inbound webhook signature. Fix: Verify with the vendor's own constructor as the handler's first statement (`stripe.webhooks.constructEvent`, `@octokit/webhooks` `verify`, Slack's `SignatureVerifier`), and use the object it returns rather than `req.body`. (OWASP A08, CWE-345)
- Gate 100 — Named fields in log records. Fix: Name the fields. Build an object literal from the fields the record needs — `{ event, email, outcome }` — and never pass the container the values arrived in. (OWASP A09, CWE-778)
- Gate 101 — Swallowed exception. Fix: Give every catch a body that logs the caught error with enough context to identify the path — an event name, the caller, the id — and rethrow when the error cannot be handled here. (OWASP A09, CWE-778)
- Gate 103 — Security events recorded. Fix: Emit one structured record per security-relevant outcome, carrying a stable `event` name, the acting principal's id, the target's id and the outcome — on the rejected branch too, because a record of refused attempts is what turns a single line into a visible pattern. (OWASP A09, CWE-778)
- Gate 107 — Log destination is collected. Fix: Log to stdout unconditionally and let the platform's collector own everything downstream — container runtimes, PaaS platforms and process supervisors already capture stdout, so the record is collected on the day it is written. (OWASP A09, CWE-778)
- Gate 110 — Lockfile-honouring install. Fix: Commit the lockfile, copy it in ahead of the install so the layer cache still works (`COPY package.json package-lock.json ./`), and call `npm ci`. (OWASP A06, CWE-1104)
- Gate 112 — Floating version specifier. Fix: Pin every direct dependency to a version, and compile the transitive set with `pip-compile` (or `uv pip compile`) from a `requirements.in` so those pin too. (OWASP A06, CWE-1104)
- Gate 114 — Non-registry source pinning. Fix: Pin the full 40-character commit SHA. For an archive URL, fetch to a file and check it against the vendor's published `sha256` (`sha256sum -c`) before installing from it — or install the vendor's registry package instead, which at least carries a version an audit tool can read. (OWASP A06, CWE-1104)
- Gate 115 — Package name provenance. Fix: Emit only names you can point at a source for — the manifest, the lockfile, or the developer's message. (OWASP A06, CWE-1104)
- Gate 117 — Audit step in the pipeline. Fix: Add the audit as its own step after the install, in the workflow that already runs the tests: `npm audit --audit-level=high`, or `pip-audit -r requirements.txt`, or `osv-scanner -r .` across a polyglot repo. (OWASP A06, CWE-1104)
- Gate 118 — Suppressed audit result. Fix: Upgrade to the fixed release. Where upstream has no fix, ignore exactly one advisory by id, with a comment stating why it is not reachable here and a date to revisit — and if the finding is real and unfixable, say that to the developer rather than making it quiet. (OWASP A06, CWE-1104)
- Gate 120 — Rate limit on credential endpoints. Fix: Attach a limiter middleware to every route that verifies or issues a credential (`express-rate-limit`, `slowapi`, the framework's own throttle class). (OWASP A04, CWE-657)
- Gate 121 — Per-account failure backoff. Fix: Store `failedAttempts` and `lockedUntil` on the user row, increment on the failure branch, set `lockedUntil` from a backoff that grows with the count, and clear both on success. (OWASP A04, CWE-657)
- Gate 123 — Result-set ceiling. Fix: Clamp the caller's value to a maximum written literally in the source, and clamp the bottom end too — `Number(req.query.limit) || 20` accepts `-5`, and a negative `take` is backwards pagination in Prisma rather than an error. (OWASP A04, CWE-657)
- Gate 125 — Request-sized work. Fix: Clamp each request-supplied magnitude to a source-literal maximum where it enters the operation, not where it is parsed — the clamp has to sit on the value the library actually receives, because a validated copy that nothing downstream reads is Gate 43. (OWASP A04, CWE-657)
- Gate 127 — Bounded expansion. Fix: Open each entry and read it through a fixed-size chunk loop, keeping a running total of bytes written that aborts the moment it crosses a source-literal ceiling. (OWASP A04, CWE-657)
- Gate 128 — Bounded quantity. Fix: Give the field an inclusive range — `.int().min(1).max(N)` — and bound the array length in the same schema, so a thousand line items of one is not the way past the per-item cap. (OWASP A04, CWE-657)
